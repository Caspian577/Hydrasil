# Hydrasil 🧬🌳

Hydrasil is a predictive data analysis system designed to assess the probability of hereditary diseases by analyzing a patient's clinical history alongside their comprehensive family medical tree.

Originally conceived to target cancer diagnostics, the project pivoted toward a broader clinical landscape, recognizing that computational genetics faces limitations in raw DNA matrix interpretation. Instead, Hydrasil leverages structured relational data to map multi-generational family medical histories and calculate long-term genetic health risks.

## 🚧 Current Status: Work in Progress (Early Stage)
This project is currently in its foundational development phase. The core back-end architecture has been successfully established:
* ✅ **Database Integration:** Active and stable connection established between the core Python environment and MySQL Workbench.
* ✅ **Environment Setup:** Basic Python structure configured to handle incoming data streams.
* ⏳ **In Development:** Expanding the full CRUD pipeline, building the multi-generational family tree mapping logic, and designing the predictive mathematical models.

## 🎯 Project Objective
To provide a scalable, data-driven solution for preventative medicine. Hydrasil processes multi-generational health data to identify risk patterns, enabling early intervention and forecasting potential future health complications based on hereditary traits.

## 🚀 Recognition & Innovation
* **Exhibited Project:** Out of numerous engineering proposals, Hydrasil was selected to be officially exhibited at the Fundación Universitaria Compensar (Unicompensar) Innovation Week project fair due to its technical architecture and real-world utility.

## 🛠️ System Architecture (Roadmap)
Once fully implemented, the application will act as a complete bridge between data analysis and database administration:
* **Database Management:** Dynamic schema management directly through the Python application using MySQL.
* **Patient Operations:** Seamless onboarding, tracking, updating, and removal of complex patient records and familial links.
* **Risk Mapping:** Connecting family trees to isolate recurring chronic conditions across generations.

## 🧰 Tech Stack
* **Language:** Python 🐍
* **Database:** MySQL Workbench 🗄️
* **Core Concepts:** Relational Database Management (RDBMS), Predictive Data Analytics, Healthcare Informatics.
